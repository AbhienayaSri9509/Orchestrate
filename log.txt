------------------------------------------------------------
Step 4: Ticket Processing Pipeline
------------------------------------------------------------
For each ticket, the pipeline executed in order:

  [1] Company detection
      → Detected company from 'company' column (HackerRank / Claude / Visa / None)

  [2] TF-IDF Retrieval
      → Retrieved top-3 corpus passages most relevant to the ticket issue
      → Computed cosine similarity score

  [3] Risk Assessment (deterministic rules)
      → Checked for high-risk keywords (fraud, billing dispute, identity theft,
        account access, security vulnerability, subscription changes, etc.)
      → Checked for prompt injection patterns
      → Checked for InfoSec / compliance form requests
      → Checked for assessment rescheduling (handled by hiring company)

  [4] Classification (keyword-based hybrid)
      → request_type : product_issue | feature_request | bug | invalid
      → product_area : domain-specific label from valid label set

  [5] Escalation Decision
      → ESCALATE if: high-risk keyword detected
                     TF-IDF confidence below threshold (< 0.15)
                     Prompt injection detected
                     Account-level action required
      → REPLY    if: corpus passage found with sufficient confidence
                     No risk flags raised

  [6] Response Generation (corpus-grounded)
      → REPLY path  : response extracted from top retrieved corpus passage
      → ESCALATE path: category-specific human-escalation template
      → No hallucinated policies — all responses sourced from corpus

  [7] Justification
      → Recorded: classification method, product area, TF-IDF score,
        escalation reason (if any), corpus document used (if replied)
